RLE Unpacker – Распаковщик RLE на 7 языках
Простая и быстрая утилита для распаковки текстовых файлов, сжатых алгоритмом Run‑Length Encoding (RLE).
Поддерживает потоковый режим, статистику восстановления и работает с любыми текстовыми данными (включая Unicode).

🚀 Возможности
Распаковка – восстанавливает исходный текст из RLE-сжатого формата.

Поддержка escape-последовательностей – корректно обрабатывает управляющие конструкции вида X\N.

Потоковая обработка – не загружает весь файл в память (в некоторых реализациях).

Гибкий ввод/вывод – чтение из файла или stdin, запись в файл или stdout.

Статистика – вывод размера исходного (сжатого) и восстановленного файла.

Цветной вывод – информативные сообщения об ошибках и успехе.

📖 Использование
Синтаксис (единый для всех языков):

bash
<команда> <входной_файл> [выходной_файл] [-v]
Аргументы
входной_файл – путь к RLE-сжатому файлу (или - для stdin).

выходной_файл – путь для сохранения распакованных данных (если не указан, вывод в stdout).

Опции
-v, --verbose – показать статистику (размер сжатого и распакованного файлов, коэффициент).

Примеры
bash
# Распаковка файла с выводом в stdout
python rle_decompress.py compressed.rle

# Распаковка с сохранением в файл
python rle_decompress.py compressed.rle restored.txt

# Чтение из stdin, запись в файл
cat compressed.rle | python rle_decompress.py - restored.txt

# С подробной статистикой
python rle_decompress.py data.rle restored.txt -v

🧠 Формат сжатых данных
Утилита ожидает файл, созданный совместимым RLE-компрессором. Формат:

Последовательности из 3 и более одинаковых символов заменяются на: символ + '\' + количество (например, A\4).

Одиночные символы и пары копируются как есть, но сам символ \ экранируется дублированием (\\).

При распаковке все \N интерпретируются как повторение предыдущего символа N раз, а \\ – как обычный обратный слеш.

Программа проверяет корректность формата и сообщает об ошибках.

✨ Дополнительные фичи
Статистика – при флаге -v выводится размер входного и выходного файлов, а также коэффициент сжатия (выходной / входной).

Обработка ошибок – детальные сообщения о некорректных escape-последовательностях.
